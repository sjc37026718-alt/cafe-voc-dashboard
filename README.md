# 로스팅하우스 온도 — 카페 VoC 자동 분류 시스템

AI(Claude) 바이브코딩으로 만든 카페 고객 피드백 수집 · 분류 · 분석 · 활용 시스템입니다.

## 무엇을 만들었나

여러 경로(앱리뷰, 설문, 전화메모, 인스타DM)에서 제각각 들어오는 고객 피드백을
하나의 입력 폼으로 표준화하고, 자동 분류된 결과를 대시보드로 시각화하며,
긍정 리뷰는 카페 홍보에 바로 활용하는 — 수집부터 분석, 활용까지 이어지는 흐름을 만들었습니다.

## 시스템 구성

```
피드백 입력 폼 → 자동 분류(불만/요청/칭찬/문의) → VoC 대시보드 → 카페 홍보 페이지
```

### 1. 피드백 입력 폼 (`feedback_form.py`)
- 날짜, 경로, 별점, 내용을 통일된 형식으로 수집
- 4개 경로의 입력 형식을 표준화하여 분류 정확도를 향상

### 2. VoC 대시보드 (`feedback_dashboard.py`)
- 급한 불만 Top 3 자동 정렬 (별점 낮은 순)
- 핵심 인사이트 3줄 자동 생성
- 유형별 · 경로별 분포 차트
- 전체 피드백 필터링 및 조회

### 3. 카페 홍보 페이지 (`cafe_page.py`)
- 칭찬 리뷰를 카페 홍보에 바로 활용

## 자동 분류 로직

| 조건 | 분류 |
|------|------|
| 별점 ≤ 2 또는 불만 키워드 포함 | 불만 |
| 요청 키워드 포함 | 요청 |
| 문의 키워드 포함 | 문의 |
| 해당 없음 | 칭찬 |

## 활용 도구

| 도구 | 역할 |
|------|------|
| Claude | 전체 코드 작성 + VoC 분류 프롬프트 설계 (바이브코딩) |
| Streamlit | 웹 대시보드 · 입력 폼 · 홍보 페이지 |
| Plotly | 차트 시각화 |
| JSON | 앱 간 데이터 공유 (DB 없이 연결) |

## 실행 방법

```bash
pip install -r requirements.txt
streamlit run feedback_dashboard.py
```

## 배운 점

- 단독 앱보다 '수집→분류→분석→활용'을 하나로 연결하면 훨씬 설득력 있는 스토리가 만들어짐
- 표면적 문제(피드백 분류) 해결 후 근본 원인(입력 표준화)까지 짚으면 차별화됨
- JSON 파일 공유만으로 여러 앱을 연결할 수 있어 DB 없이도 데모가 가능
