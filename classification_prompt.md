# VoC 분류 프롬프트 & 로직

## 1. 초기 분류 프롬프트 (Claude에게 직접 요청)

```
이 CSV의 각 의견을 유형(불만/요청/칭찬/문의)으로 분류해줘.

분류 기준:
- 불만: 부정 경험, 문제 제기, 별점 1~2점
- 요청: 새 기능/개선 희망
- 칭찬: 긍정 경험, 만족 표현
- 문의: 질문, 정보 요청

예시(Few-shot):
"라떼가 너무 달아요. 당도 조절 옵션 있으면 좋겠어요." → 불만
"비건 디저트도 있나요? 우유 알레르기가 있어서요." → 문의
"케이크 퀄리티가 베이커리급이에요. 강력 추천!" → 칭찬
"시즌 한정 메뉴 자주 나왔으면 좋겠어요." → 요청

이제 아래 CSV 데이터를 같은 기준으로 분류해서 '유형' 열을 추가해줘.
```

## 2. 급한 불만 정의

> 별점 1점 + 금전 피해·운영 품질 직결 이슈를 최우선 불만으로 정의

우선순위 점수: 별점이 낮을수록 급함 (별점 없음 = 2점으로 간주)

## 3. 자동 분류 로직 (cafe_page.py 입력 폼)

```python
# 별점 기반
별점 ≤ 2 → 불만

# 키워드 기반
불만 키워드: 불만, 오류, 느려, 오래, 잘못, 안돼, 불편, 식었, 끊겨, 달아, 좁아, 부담, 환불
요청 키워드: 있으면, 좋겠, 해주세요, 가능한가, 되나요, 해주셔
문의 키워드: 어떻게, 몇 시, 되나요, 찾으러, 할인
그 외:      칭찬

# 판정 순서
1. 불만 키워드 포함 OR 별점 ≤ 2 → 불만
2. 요청 키워드 포함 → 요청
3. 문의 키워드 포함 → 문의
4. 해당 없음 → 칭찬
```
